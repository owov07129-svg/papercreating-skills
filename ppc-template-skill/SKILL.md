# PPC Template Skill — 模板分析与排版大纲

读取用户提供的论文模板和论文内容，生成详细的论文排版大纲。

## 输入检查与主动询问

### 入口判断（在执行 Phase 1 之前必须完成）

触发本技能时，先判断以下三种情况：

```
用户输入中是否包含模板文件？
├── 是 → 进入 Phase 1（正常流程）
└── 否 → 情况 A

用户是否提出了排版要求（即使没有模板）？
├── 是 → 进入 Phase 0-B（仅要求模式）
└── 否 → 情况 B
```

### 情况 A：有论文内容但无模板

主动询问：

> **"你没有提供论文模板。请选择以下方式之一：**
>
> **a) 提供模板文件** — 上传 `.tex`、`.docx` 或 `.pdf` 格式的论文模板
>
> **b) 列出排版要求** — 告诉我以下信息（知道多少说多少）：
> - 论文类型（学位论文 / 期刊投稿 / 会议论文）
> - 目标期刊或学校（如有格式规范我会尽量遵循）
> - 纸张大小（A4 / US Letter）
> - 分栏（单栏 / 双栏）
> - 参考文献格式（GB/T 7714 / APA 7 / IEEE / Chicago / MLA / ...）
> - 字号偏好（如"正文小四"、"正文 12pt"）
> - 行距偏好（如"1.5 倍行距"、"固定 22pt"）
>
> **c) 使用内置默认模板** — 我会用通用学术排版规范（A4 / 单栏 / 宋体小四 1.5 倍行距 / GB/T 7714），你可以之后再调。"**

### 情况 B：既无模板也无要求

主动询问：

> **"你没有提供论文模板，也没有说明排版偏好。在开始之前，请先回答以下问题：**
>
> **1. 论文类型？**
> - 学位论文（本科 / 硕士 / 博士）
> - 期刊投稿（目标期刊是？）
> - 会议论文（目标会议是？）
>
> **2. 格式要求你了解的有哪些？**（不知道的可以跳过）
> - 纸张大小？
> - 单栏还是双栏？
> - 字号大小？
> - 行距？
> - 参考文献格式（GB/T 7714 / APA 7 / IEEE / Chicago / MLA / ...）？
>
> **3. 学校/期刊有官方模板吗？**（有的话请提供文件）
>
> **4. 输出格式偏好？**（LaTeX / DOCX / PDF / 都可以）"

### Phase 0-B：仅要求模式（有要求无模板）

当用户只有口头描述的排版要求而无模板文件时：

1. 将用户的文字描述转换为排版规范条目
2. 每一项标注来源：`[用户指定]` vs `[推断默认]`
3. 推断默认项在排版大纲中加 ⚠️ 标记，提醒用户确认
4. 生成排版大纲时，"待确认项"一节列出所有推断默认项

### 模板输入提示

如果用户提供的是 PDF 模板，追加提示：

> **"PDF 模板分析有一定局限（字号、行距可能不精确）。你是否有该模板的 .tex 或 .docx 源文件？提供源文件会让排版分析更准确。"**

## 输入要求（正常流程）

当用户已提供材料时，确认以下三项齐全：

1. **论文模板文件**（必须）— `.tex` / `.docx` / `.pdf` 格式均可
2. **论文内容**（必须）— 可以是：
   - 已生成的论文文本（Markdown / Word / 纯文本）
   - `.tex` 源文件
   - 结构化大纲
3. **用户补充要求**（可选）— 如"参考文献用 GB/T 7714"、"不要分栏"、"图表统一放文末"等

## 执行流程

### Phase 1：模板排版扫描

根据模板格式选择扫描方式：

#### 如果是 `.tex` 模板

直接读取 `.tex` 源文件，析取以下信息：

```
文档类参数：\documentclass[...]{...}  → 字号、纸张、分栏
宏包分析：   \usepackage{...}          → 功能需求清单
页面设置：   \geometry{...}            → 页边距
行距：       \linespread{...}          → 行距倍数
标题格式：   \titleformat{...}         → 标题字体/字号/颜色
章节格式：   \ctexset{...}             → 中文标题格式
页眉页脚：   \fancyhead{...} \fancyfoot{...} → 页眉页脚内容
字体设置：   \setmainfont{...} \setCJKmainfont{...} → 中英文字体
表格：       \begin{table}...          → 表格环境和样式
图片：       \includegraphics[...]{...} → 图片插入方式和默认参数
交叉引用：   \ref{} \cite{} \autoref{} → 引用宏包和格式
参考文献：   \bibliography{...} 或 \printbibliography → 参考文献样式
```

#### 如果是 `.docx` 模板

用 `python-docx` 读取，分析：

```python
import docx
doc = docx.Document("template.docx")

# 页面设置
for section in doc.sections:
    print(f"页边距: 上{section.top_margin} 下{section.bottom_margin} "
          f"左{section.left_margin} 右{section.right_margin}")
    print(f"页面大小: {section.page_width} × {section.page_height}")

# 样式扫描
for style in doc.styles:
    if style.type == docx.enum.style.WD_STYLE_TYPE.PARAGRAPH:
        font = style.font
        pf = style.paragraph_format
        print(f"样式: {style.name}")
        print(f"  字体: {font.name} {font.size} 粗体={font.bold}")
        print(f"  行距: {pf.line_spacing} 段前={pf.space_before} 段后={pf.space_after}")
        print(f"  首行缩进: {pf.first_line_indent}")
```

#### 如果是 `.pdf` 模板

用 `PyMuPDF` + `pdfplumber` 分析：

```python
import fitz
doc = fitz.open("template.pdf")
page = doc[0]
# 字体列表
fonts = page.get_fonts()
# 文本块位置和字号
blocks = page.get_text("dict")["blocks"]
```

同时建议用户提供 `.tex` 或 `.docx` 源文件以提高分析精度。

### Phase 2：排版细节清单

按以下 18 个维度逐项记录，无法确定的项目标注"未检测到，需人工确认"：

| # | 维度 | 检测内容 | 示例值 |
|---|------|----------|--------|
| 1 | **页面尺寸** | 纸张大小（A4 / US Letter / 自定义） | A4 |
| 2 | **页边距** | 上下左右边距 | 上 2.5cm 下 2.0cm 左 3.0cm 右 2.5cm |
| 3 | **分栏** | 是否分栏、栏数、栏间距 | 单栏 / 双栏 0.8cm |
| 4 | **标题字体** | 论文标题的字体、字号、加粗、颜色 | 黑体 小二 加粗 |
| 5 | **章节标题字体** | 一级/二级/三级标题字体字号 | 一级：黑体 三号；二级：黑体 四号 |
| 6 | **正文字体** | 中文字体、英文字体、字号 | 中文宋体 小四；英文 Times New Roman 12pt |
| 7 | **行间距** | 正文行距（倍数或固定值） | 1.5 倍 / 固定 22pt |
| 8 | **段落间距** | 段前、段后间距 | 段前 0 段后 6pt |
| 9 | **首行缩进** | 是否缩进、缩进量 | 2 字符 |
| 10 | **页眉** | 页眉内容、字体、字号、横线 | 奇数页：论文题目；偶数页：章节名 |
| 11 | **页脚/页码** | 页码位置、格式 | 底部居中 阿拉伯数字 |
| 12 | **表格格式** | 表格宽度、边框样式、标题位置、字号 | 三线表 表题在上 五号 |
| 13 | **表注/脚注** | 表注位置、字体、格式 | 表下 小五号 |
| 14 | **图片** | 默认宽度、标题位置、字号 | 宽度=\textwidth 图题在下 五号 |
| 15 | **公式** | 公式编号方式、编号位置 | 右对齐 (章节-序号) |
| 16 | **参考文献** | 引用格式（著录规则）、悬挂缩进 | GB/T 7714-2015 顺序编码制 |
| 17 | **页面背景/装饰** | 水印、边栏、校徽 | 无 |
| 18 | **上标/下标** | 作者标注、脚注标记格式 | 作者：† 通讯作者；脚注：①②③ |

### Phase 3：论文内容阅读

读取用户提供的论文内容，提取：

```
标题：
作者：
单位：
摘要（中/英）：
关键词（中/英）：
目录（如已生成）：
正文（按章节）：
  ├── 1. 引言
  ├── 2. 文献综述 / 相关工作
  ├── 3. 方法
  ├── 4. 实验 / 验证
  ├── 5. 讨论
  └── 6. 结论
图表列表：
  ├── 图1: xxx
  ├── 表1: xxx
  └── ...
公式列表：
  ├── 公式1: xxx
  └── ...
参考文献列表（共 N 条）：
附录（如有）：
声明（如有）：
```

### Phase 4：用户要求整合

询问用户：
1. 输出格式？`LaTeX` / `DOCX` / `PDF` / `LaTeX+PDF`
2. 参考文献格式？`GB/T 7714` / `APA 7` / `IEEE` / `Chicago` / `MLA` / 跟随模板
3. 语言？`中文` / `中英双语` / `英文`
4. 有无特殊要求？（页数限制、特定期刊命名规范、图表色彩要求等）

### Phase 5：生成排版大纲

将 Phase 1-4 的信息整合为结构化的"论文排版大纲"，格式如下：

```markdown
# [论文标题] — 排版大纲

## 一、页面基础
- 纸张：A4
- 页边距：...
- 分栏：...
- 行距：...

## 二、字体体系
- 中文正文：...
- 英文正文：...
- 一级标题：...
- ...

## 三、段落与间距
- ...

## 四、页眉页脚
- ...

## 五、图表公式规范
- ...

## 六、参考文献格式
- ...

## 七、内容与结构映射
- [章节1 标题] → [模板 Section 样式]
- [章节2 标题] → [模板 Subsection 样式]
- ...

## 八、待确认项
（Phase 2 中无法确定的项目列在这里，标注来源：[推断默认] / [需人工量]）

## 九、用户特殊要求
（用户提出的个性化要求）

## 十、LaTeX 实现清单
\documentclass[...]{...}
\usepackage{...}    # 每个宏包注明用途
\geometry{...}
\ctexset{...}
...
```

### Phase 6：导出大纲

将排版大纲保存为 `排版大纲.md`，放在用户指定的工作目录。同时输出到对话中供用户审阅。

---

## 关键规则

1. **宁可标注"未检测到"也不要猜**——排版信息错了后面全错
2. `.docx` 模板优先用 `python-docx` 的 styles API，比视觉推断准确
3. `.pdf` 模板分析结果标注置信度：高（字体直接编码）/ 中（位置推断）/ 低（需人工量）
4. 论文内容中未出现的元素（如表注、附录），在排版大纲中标注"论文中未出现，保留模板规范"
5. 排版大纲必须经用户确认后才传给 ppc-edit-skill
